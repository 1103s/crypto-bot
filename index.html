get_market_range(id, vs_currency, days)#

Granularity is automatically determined by Coingecko using the following specifications:

5-minute intervals: 1 day from query time hourly intervals: 1-90 days from query time daily intervals: More than 90 days from query time

Use unix UTC for start and end

Parameters

  • id (str) – A valid Coingecko symbol string

  • vs_currency (str) – A valid quote currency (e.g. USD)

  • end (int) – The ending time in the range (UTC Unix Timestamp)

Returns

Returns spot price/volume data in json format if coin_id is valid. Otherwise, None is returned.

Return type

json

get_ohlc(id, vs_currency, days)#

Coingecko’s OHLC API for OHLC data. The time step intervals are automatically set by Coingecko as follows:

1-2 days: 30 minute intervals 2 < days <= 30: 4 hour intervals

Parameters

  • id (str) – The Coingecko coin ID (ethereum, litecoin, etc.)

  • vs_currency (str) – The currency to weigh the coin against (usd, eur, etc.)

Returns

Timestamp (ms), Open, High, Low, Close in a numpy array

ML_predictor_backend.py#

Library code for the Machine Learning prediction backend

ML_predictor_backend.predict_next_N_timesteps(data, lags, N, model_name)#

Forecasts price data given an input of historical price data, and a ML regression model to do the prediction.

Parameters

  • data – Input time-ordered historical price data to predict on

  • lags – Integer hyperparamater for the forecasting prediction

  • N – Integer number of time steps (days) in the future to predict the price

  • model_name – String machine learning model name. Corresponds to a sklearn regression algorithm.

Returns

List of time-ordered price predictions. Exits if the specified model_name is not recognized.

Return type

list

utils.plot_and_save_price_graph(data, filename, file_extension, crypto)#

Saves the historical price data as an image file in figures/

Parameters

  • data – Time ordered list of prices to plot (assumed to be in USD based on the API sources)

  • filename – Filename under which to save the data in the directory figures/

  • file_extension – file extension to save the image as. Either png, jpg, or pdf

  • crypto – The name of cryptocurrency being plotted. Typically the symbol of the cryptocurrency (e.g. BTC)

Returns

None

utils.plot_and_save_price_graph_with_predictions(data, filename, file_extension, crypto, predictions)#

Saves the prediction price data as an image file. 50 days of historical data is also plotted to show in the near term where the prediction trends are going. If there are multiple ML model predictions, all of the model predictions are plotted. Image is saved to figures/

Parameters

  • data – Time ordered list of prices to plot (assumed to be in USD based on the API sources)

  • filename – Filename under which to save the data in the directory figures/

  • file_extension – file extension to save the image as. Either png, jpg, or pdf

  • crypto – The name of cryptocurrency being plotted. Typically the symbol of the cryptocurrency (e.g. BTC)

  • predictions – Dictionary where the keys are ML model names (e.g. random_forest) and the values are lists of time ordered price predictions

Returns

None
